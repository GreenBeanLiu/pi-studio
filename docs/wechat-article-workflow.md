# 微信订阅号文章工作流

当前版本先生成可审阅的公众号草稿，不直接调用微信发布接口。

工作流编辑器提供节点库；“快速模板”只是把常用节点预先排好，保存后仍可删除、添加和调整顺序。

## 流程

1. 资料与事实：围绕 Brief 检索资料，保留来源和不确定项。
2. 文章大纲：输出标题候选、摘要、开头钩子、分节和 CTA。
3. 公众号初稿：生成适合手机阅读的中文正文，控制段落长度，避免夸大表达。
4. 事实与合规审校：检查来源、广告法风险、绝对化表述和错别字，并给出修订稿。
5. 保存公众号草稿：把上一步完整输出写为工作区内的 Markdown 或 HTML 文件。
6. 发送预览提醒：通知渠道只发送文件路径，人工确认后再复制到公众号后台。

## 可选节点

- 文章：资料与事实、文章大纲、公众号初稿、人工审核、事实与合规审校。
- 媒体：公众号封面图、通用生图。
- 输出：公众号 HTML、Markdown、预览提醒。
- 通用：通用智能体、通用导出、通用通知。

## 变量和产物

- `{{routine.input}}`：本次文章 Brief。
- `{{prev.output}}`：上一步的文本产物（单步最多保留 60000 字）；运行记录中的摘要仍会截断到 4000 字。
- `{{steps.<步骤名>.output}}`：指定步骤的文本产物。
- `{{steps.<步骤名>.imageUrl}}`：指定生图步骤的公网图片链接。
- 导出节点的路径必须位于当前工作区内；HTML 节点把上一步的 Markdown 转换成公众号正文片段，不包含微信 API 的封面 media_id。

## 运行边界

- 定时 Agent 节点会复用设置里的 Docker 沙箱开关；沙箱开启但 Docker 或对应 pi 镜像未就绪时，任务会失败并记录原因。
- 当前“通知”节点支持本地、Webhook 和飞书，不等于微信发布。
- “人工审核”节点会暂停当前运行并弹出草稿预览；通过后才会继续后续导出/通知节点，拒绝、停用、删除或 30 分钟无响应都会停止本次运行。
- 微信草稿箱适配器仍未接入，默认不自动发布。
