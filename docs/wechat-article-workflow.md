# 微信订阅号文章工作流

当前版本支持“读取本地素材 → 事实梳理 → 写正文 → 人工审核 → 封面/正文配图 → 存飞书文档并创建微信公众号草稿”的流程，默认不自动群发。

工作流编辑器提供节点库；“快速模板”只是把常用节点预先排好，保存后仍可删除、添加和调整顺序。

## 流程

1. 读取素材文件夹：递归读取工作区内的文本、Markdown、HTML、CSV、JSON、YAML 和常见图片；留空时跳过。
2. 资料与事实：结合本地素材和联网检索，保留来源和不确定项。
3. 文章大纲：输出标题候选、摘要、开头钩子、分节和 CTA。
4. 公众号初稿：生成适合手机阅读的中文正文，控制段落长度，避免夸大表达。
5. 人工审核：确认正文后才继续生成和上传图片。
6. 封面与正文配图：既可以使用素材文件夹图片，也可以由生图节点补齐。
7. 微信公众号草稿：上传最终图片、转换正文并创建草稿。

## 可选节点

- 文章：资料与事实、文章大纲、公众号初稿、人工审核、事实与合规审校、飞书文档、微信公众号草稿。
- 媒体：公众号封面图、通用生图。
- 输出：公众号 HTML、Markdown、飞书文档、微信公众号草稿、预览提醒。
- 通用：通用智能体、通用导出、通用通知。

## 变量和产物

- `{{routine.input}}`：本次文章 Brief。
- `{{prev.output}}`：上一步的文本产物（单步最多保留 60000 字）；运行记录中的摘要仍会截断到 4000 字。
- `{{steps.<步骤名>.output}}`：指定步骤的文本产物。
- `{{steps.<步骤名>.imageUrl}}`：指定生图步骤的公网图片链接。
- “读取素材文件夹”节点的文本会通过 `{{steps.本地素材.output}}` 传给后续节点；图片作为结构化素材随运行上下文传递，不写入提示词。
- 导出节点的路径必须位于当前工作区内；HTML 节点把上一步的 Markdown 转换成公众号正文片段，不包含微信 API 的封面 media_id。

## 运行边界

- 定时 Agent 节点会复用设置里的 Docker 沙箱开关；沙箱开启但 Docker 或对应 pi 镜像未就绪时，任务会失败并记录原因。
- 当前“通知”节点支持本地、Webhook 和飞书，不等于微信发布；微信公众号渠道只能由“微信公众号草稿”节点使用。
- 飞书通知使用 interactive 卡片；飞书应用渠道可配置 `folderToken`，文档会创建到共享文件夹。
- 配图会按 Image Block 上传到文档；来源 URL 会转换为可点击链接。
- 飞书应用需要开通 `docx:document`、`drive:drive` 等文档/云空间权限；使用 `tenant_access_token` 时，文件夹通常必须是应用创建或应用可管理的文件夹。
- 旧版“微信公众号文章生成”工作流启动时会自动补上“存飞书文档”节点，不会修改其他自定义工作流。
- “人工审核”节点会暂停当前运行并弹出草稿预览；通过后才会继续后续导出/通知节点，拒绝、停用、删除或 30 分钟无响应都会停止本次运行。
- 微信公众号渠道使用 AppID/AppSecret 获取 access_token。名称含“封面”或 `cover` 的图片优先作为封面；没有显式封面时优先使用第一张生成图。其余素材图片和生成图片去重后全部上传为正文图，再调用草稿箱接口；草稿节点只创建草稿，不调用发布/群发接口。
